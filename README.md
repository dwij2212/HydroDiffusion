## Diffusion ssm for streamflow forecasting

---

### Repository Structure

```
├── train.sh          # Training script wrapper
├── test.sh           # Evaluation script wrapper
```

## Prerequisites

* Python 3.8+
* PyTorch 2.x with CUDA support
* NVIDIA GPU (e.g., L40S, RTX 4090)
* Required Python packages:

  ```bash
  pip install -r requirements.txt
  ```

## Setting the GPU device

By default, you can select which GPU to use by setting the `CUDA_VISIBLE_DEVICES` environment variable. For example, to expose only GPU 0:

```bash
export CUDA_VISIBLE_DEVICES=0
```

Or inline:

```bash
CUDA_VISIBLE_DEVICES=0 ./train.sh <model> <static|no_static> 0 <run_dir> <note>
```

## Training: `train.sh`

### Usage

```bash
./train.sh <model> <static|no_static> [gpu] [run_dir] [note]
```

* **model**: Model name, one of `seq2seq_lstm`, `encdec_lstm`, `seq2seq_ssm`, `diffusion_lstm`, `diffusion_unet`, `diffusion_ssm`.
* **static|no\_static**: Whether to use static features (`static`) or not (`no_static`).
* **gpu**: (Optional) CUDA device ID (default: 0).
* **run\_dir**: Directory where logs and checkpoints are saved.
* **note**: (Optional) Tag appended to log file names.

### Example

```bash
mkdir -p runs/run_diffusion_ssm
./train.sh diffusion_ssm static 0 runs/run_diffusion_ssm myExperiment
```

This will:

1. Set `CUDA_VISIBLE_DEVICES=0`.
2. Launch training with the `diffusion_ssm` model using static features.
3. Save logs to `reports/test_diffusion_ssm_static_<seed>_myExperiment.out`.

---

## Evaluation: `test.sh`

### Usage

```bash
./test.sh <model> <static|no_static> [gpu] [run_dir] [note]
```

Arguments are the same as `train.sh`. This script runs the evaluation (`main.py evaluate`) and writes outputs to log files.

### Example

```bash
./test.sh diffusion_ssm static 0 runs/run_diffusion_ssm myExperiment
```

## Key Configuration Flags

* `--model_name`: Specifies which model to train/evaluate.
* `--gpu`: GPU index (as seen inside the container, typically `0` when `CUDA_VISIBLE_DEVICES` is set).
* `--no_static` / `--concat_static`: Controls static feature inclusion.
* `--run_dir`: Directory for model outputs and checkpoints.
* **Diffusion-specific flags** (only for `diffusion_ssm`):

  * `--epochs`, `--batch_size`, `--d_model`, `--lr`, `--weight_decay`, etc.

Refer to `papercode/train_generic.py` for full list.

## Freeing GPU Memory

If training or evaluation processes exit unexpectedly and GPU memory remains allocated, you can free it by:

```bash
pkill -u $USER pt_data_w  # kills PyTorch data-loader workers
pkill -u $USER python3    # kills stray Python processes
```

Then verify with:

```bash
nvidia-smi -i <GPU_INDEX>
```

## Troubleshooting

* **Dimension mismatch**: If you see `mat1 and mat2 shapes cannot be multiplied`, ensure your `--d_model` matches the architecture's expected embedding dimension (e.g., 256).
* **CUDA errors**: Check `CUDA_VISIBLE_DEVICES` and confirm the device ID exists and is not in use.
