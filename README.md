## Training: `train.sh`

### Usage

```bash
./train.sh <model> <static|no_static> [gpu] [note]
```
* **model**: Model name,`encdec_lstm` (deterministic), `seq2seq_ssm`, `diffusion_lstm`, `diffusion_unet`, `diffusion_ssm`.
* **static|no\_static**: Whether to use static features (`static`) or not (`no_static`).
* **gpu**: (Optional) CUDA device ID (default: 0).
* **note**: (Optional) Tag appended to log file names.

### Example

```bash
./train.sh diffusion_ssm static 0 myExperiment
```

This will:

1. Set `CUDA_VISIBLE_DEVICES=0`.
2. Launch training with the `diffusion_ssm` model using 27 static features.
3. Save logs to `reports/test_diffusion_ssm_static_<seed>_myExperiment.out`.

---

## Testing: `test.sh`

### Usage

```bash
./test.sh <model> <static|no_static> [gpu] [run_dir] [note]
```

Arguments are the same as `train.sh`. This script runs the evaluation (`main.py evaluate`) and writes outputs to log files.

To find the run_dir: go to the train log of the experiment you want to test. You can find the run_dir in the line of "Run directory is: [run_dir]".

### Example

```bash
./test.sh diffusion_ssm static 0 [run_dir] myExperiment

## Evaluation: `analysis/main_performance_ensemble_only.py`

Within `main_performance_ensemble_only.py`, specify the `run_dir` of the model for evaluation (i.e., evaluation metrics calculation).

### Usage

```bash
python analysis/main_performance_ensemble_only.py <performance_metrics_output.csv>
```

The script will load the ensemble results from the specified `run_dir`, calculate performance metrics per lead-time, and save them as a CSV file (`<performance_metrics_output.csv>`).
